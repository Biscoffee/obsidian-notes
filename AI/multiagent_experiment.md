## 结论（亲手验证『要不要 multi-agent』）

- **多 agent 整体更优**：B 比 A 平均 +1.33 分，代价是 token 4.8x、耗时约 3.1x。
- 有 0/12 题批评家判「无重大问题」直接跳过修订——说明**不是每题都需要多 agent**，简单题上多 agent 是纯浪费。
- 印证学习路线的反模式警告：multi-agent 的增益要和它的 token/延迟成本对比着看，**先把单 agent 做好，证明不够再上**。

批评家意见（节选）：1.  **事实/原理错误**：在“四、 结论与结合建议”的结合路径1中，将长上下文模型称为“生成器”是技术描述不准确。它在此角色中更准确地说是“处理器”或“阅读器”，其核心能力是**理解并整合**已检索到的多个上下文片段，最终的生成仍依赖该模型。称其为“生成器”可能误导读者。
