为什么要拆出 rwe — WWDC 2020 内存优化
旧方案（objc4-781 之前）
class_ro_t
class_rw_t（全部类）
每个类都分配 class_rw_t
含完整 methods / props / protos
典型 App ≈ 30MB 用于 rw
新方案（objc4-781+）
class_ro_t
class_rw_t（精简，无 methods）
rwe（仅10%）
rw 不再存方法列表，只存指针
90% 的类不创建 rwe
节省 ≈ 14MB（Apple 数据）
class_ro_t：编译期写入 __DATA_CONST，只读，包含 ivars / baseMethods / name 等，运行时永远不变。   class_rw_t：realizeClass() 时创建，通过 ro_or_rwe 低位 bit 区分当前指向 ro 还是 rwe，methods() / properties() 等访问器对调用方透明。   class_rw_ext_t：仅在 Category 附加或运行时修改方法时按需创建（约 10% 的类），持有合并后的完整方法列表并保存一份 ro 指针。鼠标悬停字段行查看详情。
